# Contributing to DLP System

Thank you for your interest in this project. This repository currently contains **architecture and design documentation** for an enterprise Data Loss Prevention (DLP) platform — there is no application source code to contribute to yet.

## Scope of Contributions

At this stage, contributions are welcome in the following areas:

- Improvements to architecture and design documentation
- Corrections to technical accuracy or terminology
- Suggestions for additional design considerations (scalability, security, compliance)
- Diagram improvements (Mermaid diagrams, architecture visuals)
- Roadmap suggestions

This repository does **not** currently accept source code contributions (frontend, backend, or agent implementation), as those components have not yet been developed.

## How to Contribute

1. Fork the repository.
2. Create a feature branch for your change (`git checkout -b docs/improve-architecture-section`).
3. Make your changes to the relevant documentation file(s) under [docs/](docs/).
4. Ensure Markdown formatting is consistent with the rest of the repository.
5. Open a pull request describing the motivation and scope of your change.

## Documentation Standards

- Use clear, professional, technically accurate language.
- Clearly distinguish between **current state**, **planned features**, and **future roadmap** items — do not describe planned functionality as implemented.
- Use tables, Mermaid diagrams, and headings consistent with existing documents in [docs/](docs/).
- Keep explanations concise and avoid marketing language or unverifiable claims.

## Reporting Issues

If you notice inconsistencies, inaccuracies, or gaps in the documentation, please open an issue describing:

- The affected document/section
- The nature of the issue
- A suggested correction, if applicable

## Code of Conduct

Please be respectful and constructive in all discussions, issues, and pull requests.
